# 29 任务执行算法、重试、幂等与补偿

## 1. 设计目标

- 相同输入多次执行不产生混乱
- 失败后能局部恢复
- 每个任务可追踪来源、参数和产物
- 大模型任务支持超时、取消和重新排队

---

## 2. 任务状态

| 状态 | 说明 |
|---|---|
| pending | 等待调度 |
| queued | 已入队 |
| running | worker 已接手 |
| succeeded | 成功 |
| failed_retryable | 失败但可重试 |
| failed_terminal | 失败且不可自动恢复 |
| cancelled | 被用户取消 |
| skipped | 因缓存命中或条件不满足被跳过 |
| blocked | 等待人工审核或上游依赖 |

---

## 3. 幂等键设计

每个任务必须有 `idempotency_key`，建议格式：

```text
{episode_id}:{stage}:{target_id}:{input_hash}:{profile_version}
```

示例：

```text
ep01:image_generate:ep01_sc02_sh05:9a23d1b7:sd35m_ipadapter_v1
```

当相同幂等键的任务已成功完成，直接返回已有产物，不重复执行。

---

## 4. 调度伪代码

```python
while True:
    ready_tasks = repo.fetch_ready_tasks(limit=20)
    for task in ready_tasks:
        if not deps_satisfied(task):
            continue
        if cache_hit(task.idempotency_key):
            mark_skipped(task)
            attach_cached_artifacts(task)
            continue
        if task.requires_gpu and not gpu_lock.available():
            continue
        dispatch(task)
```

## 5. 重试策略矩阵

| 阶段 | 最大重试 | 退避策略 | 重试前动作 |
|---|---:|---|---|
| llm_generate | 2 | 5s, 20s | 降低输出长度或切 profile |
| tts_generate | 2 | 3s, 10s | 切换 style tag / 断句 |
| image_generate | 3 | 10s, 30s, 60s | 切 seed / 降 steps / 替换 control ref |
| video_generate | 2 | 20s, 60s | 降帧数 / 降分辨率 / 重写 motion prompt |
| compose | 1 | 5s | 重新生成 concat list |
| qc_check | 1 | 5s | 切轻量检查模式 |

## 6. 补偿动作

### 6.1 TTS 成功但数据库写入失败

补偿：
- 扫描任务输出目录
- 按 checksum 重建 artifact
- 回填 `tasks.outputs`

### 6.2 图像已生成但 worker 崩溃

补偿：
- 若目标路径文件存在且通过摘要校验
- 标记 `task` 为 `succeeded_with_recovery`

### 6.3 视频阶段中断

补偿：
- 单镜头为最小补偿单元
- 已完成镜头不重跑
- 未完成镜头回 `pending`

---

## 7. 取消策略

用户取消分三类：

1. **取消章节级 job**：未开始任务标记 cancelled，进行中的任务发送 stop 信号
2. **取消单镜头**：只影响该 `shot_id`
3. **取消后重排**：保留已有产物，但新建一个 job version

---

## 8. 失败分类

### 可自动恢复
- OOM
- 模型加载超时
- 网络类错误（仅限本地服务内部 HTTP 调用）
- FFmpeg 临时文件缺失

### 需要人工处理
- 脚本逻辑错误
- 角色风格漂移严重
- TTS 发音与角色设定冲突
- 视频内容与镜头意图不符

### 终止性错误
- 输入 schema 非法
- 引用资产不存在
- 配置文件损坏
- 磁盘空间不足且不可清理

---

## 9. 任务执行日志要求

每个 task 必须记录：

- 任务入参摘要
- 依赖 artifact id
- worker profile
- seed / steps / cfg / resolution / frame_count
- GPU 开始和结束时间
- 标准输出尾部
- 失败分类和 stack trace 摘要

---

## 10. 断点恢复原则

- `scene` 不作为恢复粒度，过大
- `shot` 是最小视频恢复粒度
- `audio_segment` 是最小音频恢复粒度
- `keyframe` 是最小图像恢复粒度
- `final compose` 只要素材齐全即可单独重跑
