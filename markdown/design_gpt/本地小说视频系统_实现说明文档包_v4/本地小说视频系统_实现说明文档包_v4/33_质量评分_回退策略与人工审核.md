# 33 质量评分、回退策略与人工审核

## 1. 质量控制目标

v1 的质量控制不是“完全自动美学判断”，而是建立实用 gate：

- 内容是否符合脚本
- 角色是否保持一致
- 语音是否可听懂且风格合理
- 视频是否出现明显畸形或闪烁
- 成片是否能顺畅播放

---

## 2. 评分层级

### 2.1 片段级

- 单句音频
- 单张关键帧
- 单镜头视频

### 2.2 章节级

- 整体节奏
- 风格一致性
- 叙事完整性

---

## 3. 指标体系

### 音频
- ASR 一致率
- 峰值电平
- 静音比例
- 语速区间
- 情绪标签匹配度（规则式）

### 图像
- 人脸可辨识度
- 角色参考相似度
- 构图与 shot framing 匹配度
- 局部解剖错误

### 视频
- 闪烁分数
- 脸部一致性
- motion prompt 匹配度
- 时长误差
- 场景连续性

---

## 4. 评分阈值建议

| 项目 | 通过阈值 |
|---|---:|
| ASR 一致率 | >= 0.96 |
| 图像角色相似度 | >= 0.80 |
| 视频 overall score | >= 0.75 |
| 时长误差 | <= 250ms |
| 音频峰值 | <= -1 dBFS |

---

## 5. 自动回退规则

### 5.1 TTS

- ASR 不一致：先重试一次
- 仍失败：切断句策略
- 仍失败：切换 voice profile 备选

### 5.2 关键帧

- 角色相似度不足：增强 IP-Adapter scale
- 构图偏离：补充 pose/depth control
- 面部崩坏：切 patch 工作流

### 5.3 视频

- 闪烁过高：重写 motion prompt，降低动作复杂度
- 时长偏差：重生成或改用尾帧冻结补齐
- 风格漂移：换用更稳定关键帧或锁定 seed family

---

## 6. 人工审核规则

只在以下情况创建 review task：

- 连续两次自动修复失败
- 关键人物 close-up 质量不达标
- 章节终检不过
- 用户主动要求审核

review task 必须附带：
- 原脚本文本
- 当前产物预览
- 本次参数与前次参数 diff
- 一键执行的 patch 选项

---

## 7. 发布 gate

章节只有在以下条件全部满足时才可发布：

- 所有必需镜头均有通过版本
- 音频总线通过质检
- 字幕文件无语法错误
- 终片可播放并完成时长检查
- 若配置了人工终审，则 review status = approved

---

## 8. 评分记录

所有评分必须落库或落 JSON：

```json
{
  "target_id": "vid_ep01_sc02_sh05_v1",
  "score_pack_version": "qc_pack_v1",
  "scores": {
    "flicker": 0.11,
    "face_consistency": 0.82,
    "motion_match": 0.76,
    "overall": 0.79
  },
  "decision": "pass"
}
```

---

## 9. 人工 patch 分类

- 文本 patch：改台词、改旁白、改镜头描述
- 图像 patch：换脸型、换服饰、换构图
- 视频 patch：改运动方式、改镜头长短
- 合成 patch：调音量、调字幕、调镜头顺序
