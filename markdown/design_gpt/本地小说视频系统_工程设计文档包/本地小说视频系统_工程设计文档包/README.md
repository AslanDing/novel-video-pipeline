## 文档边界

本套文档聚焦：

- 软件工程化落地
- 24GB 单卡本地可运行
- 模块拆分明确
- 接口与数据结构明确
- 支持迭代开发和局部重跑

本套文档不包含：

- 具体模型权重下载地址
- 第三方模型许可证逐条法务解读
- 前端设计稿
- 商业化运营方案

---

## 推荐 MVP 路线

优先实现以下最小闭环：

1. 用户输入构思
2. LLM 生成故事圣经、章节正文、分镜脚本
3. TTS 生成旁白与对白
4. 文生图生成关键帧
5. 图生视频生成 3–5 秒镜头
6. FFmpeg 合成最终视频
7. 支持失败镜头局部重跑

## 术语表

- **Story Bible**：故事圣经，包含世界观、人物关系、时间线和约束规则
- **Episode Outline**：章节大纲
- **Shot**：镜头级别的最小可生成单元
- **I2V**：Image-to-Video，图生视频
- **Prompt Rewrite**：将自然语言文本改写为适合图像或视频模型的提示词
- **Artifact**：过程产物，例如章节正文、音频、关键帧、视频片段
- **Orchestrator**：流程编排器，负责全流程状态和调度
- **GPU Lock**：单 GPU 互斥锁，确保重模型串行占用显存
