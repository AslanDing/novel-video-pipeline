# 05 数据模型与目录规范

## 5.1 设计原则

### 原则 1：文件系统保存大对象
音频、图像、视频、字幕等二进制或大文本内容存文件系统。

### 原则 2：数据库保存索引与关系
项目、任务、产物、版本、依赖关系等保存到数据库。

### 原则 3：结构化 JSON 优先
在模块间通信时，优先使用结构化 JSON / JSONL，而不是自由文本。

## 5.3 命名规范

### 项目级
- `project_id`: `proj_YYYYMMDD_xxxxxx`

### 章节级
- `episode_id`: `ep01`, `ep02`

### 场景级
- `scene_id`: `S01`, `S02`

### 镜头级
- `shot_id`: `S01_SH03`

### 产物级
- `artifact_id`: `art_xxxxxx`

### 任务级
- `task_id`: `task_xxxxxx`

## 5.4 核心数据库表

## projects
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 项目 ID |
| name | string | 项目名 |
| status | string | 项目状态 |
| language | string | 语言 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

## project_configs
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 配置 ID |
| project_id | string | 项目 ID |
| config_json | json | 当前项目配置 |
| version | int | 版本 |

## tasks
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 任务 ID |
| project_id | string | 项目 ID |
| task_type | string | 任务类型 |
| status | string | 状态 |
| parent_task_id | string/null | 父任务 |
| input_hash | string | 输入签名 |
| retry_count | int | 重试次数 |
| created_at | datetime | 创建时间 |

## task_runs
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | run ID |
| task_id | string | 任务 ID |
| started_at | datetime | 开始时间 |
| finished_at | datetime | 结束时间 |
| worker_name | string | 执行 worker |
| log_path | string | 日志路径 |
| error_message | text | 错误摘要 |

## artifacts
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 产物 ID |
| project_id | string | 项目 ID |
| artifact_type | string | 产物类型 |
| logical_key | string | 逻辑键，例如 ep01/script |
| version | int | 版本号 |
| state | string | active/draft 等 |
| file_path | string | 文件路径 |
| meta_json | json | 元数据 |

## artifact_dependencies
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 主键 |
| upstream_artifact_id | string | 上游产物 |
| downstream_task_id | string | 下游任务 |

## qc_reports
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 报告 ID |
| project_id | string | 项目 ID |
| target_artifact_id | string | 目标产物 |
| report_type | string | ASR/format/render |
| report_json | json | 报告内容 |

## 5.6 Artifact 元数据建议

每个 artifact 的 `meta_json` 至少包含：

```json
{
  "source_task_id": "task_001",
  "model_name": "qwen3-14b-instruct-awq",
  "model_version": "2026-03",
  "prompt_template_version": "v3",
  "input_hash": "sha256:...",
  "created_at": "2026-03-29T10:00:00Z",
  "tags": ["ep01", "shot", "keyframe"]
}
```

## 5.7 版本管理策略

### 文本资产
- 小改动：新版本号
- 人工编辑：记录 `edited_by_human = true`

### 图像资产
- 每次重生图必须递增版本
- 保留 prompt 与 seed

### 视频资产
- 保留关键帧来源
- 保留 motion prompt
- 保留 duration 和 fps

---

## 5.8 缓存键设计

推荐缓存键：

`cache_key = hash(task_type + input_hash + config_hash + model_version + template_version)`

命中时直接复用产物。

## 5.9 Schema 校验建议

建议所有结构化文件都用 Pydantic 模型校验。  
关键对象：
- StoryBible
- CharacterBible
- EpisodeOutline
- ScriptLine
- AudioArtifactMeta
- ImageArtifactMeta
- VideoArtifactMeta
- QCReport

---

## 5.10 保留与清理策略

### 永久保留
- active 产物
- 对应元数据
- render report
- 最终发布版本

### 可清理
- 失败中间缓存
- deprecated 大体积视频
- 低质量预览图

建议增加：
- `archive` 目录
- 自动清理脚本
