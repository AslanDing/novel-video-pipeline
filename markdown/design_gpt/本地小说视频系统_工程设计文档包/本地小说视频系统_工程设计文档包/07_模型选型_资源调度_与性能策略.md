# 07 模型选型、资源调度与性能策略

## 7.1 选型原则

1. 单 GPU 24GB 可运行
2. 中文能力优先
3. 支持局部重跑
4. 生态成熟、可替换
5. 能与工程编排良好结合

---

## 7.2 推荐模型栈

## 7.2.1 文本层
主力：
- Qwen3-14B-Instruct-AWQ

用途：
- 故事圣经
- 角色圣经
- 章节大纲
- 正文生成
- 分镜脚本
- prompt rewrite

不建议作为常驻主力：
- 30B 级模型  
原因：24GB 单卡持续运行压力更大，留给图像/视频阶段的空间变小。

### 备选方案
- 需要更轻量时：Qwen 7B
- 需要更强结构化时：本地双模型策略（7B 规划 + 14B 润色）

---

## 7.2.2 TTS 层
主力：
- Fish Audio S2 Pro

适合：
- 多角色
- 情绪化台词
- 戏剧性旁白

备选：
- CosyVoice 3.0

适合：
- 中文发音修正
- 数字与专有名词
- 发音更强控制需求

---

## 7.2.3 图像层
主力：
- Stable Diffusion 3.5 Medium
- IP-Adapter
- ControlNet

设计思路：
- SD3.5 负责主图像生成
- IP-Adapter 负责角色一致性
- ControlNet 负责构图与姿态控制

备选：
- FLUX dev 系列（仅在许可证允许时）

---

## 7.2.4 视频层
主力：
- HunyuanVideo-1.5 I2V 480p

推荐做法：
- 以关键帧驱动的短镜头为主
- 每个镜头 3–5 秒
- 失败镜头降级重跑

备选：
- 更轻量的 I2V 路线
- 更保守的动画镜头模板

---

## 7.3 GPU 调度原则

### 单 GPU 原则
任何时刻只允许一个重模型独占 GPU。

重模型包括：
- 大上下文 LLM 推理
- 文生图 / 图生图
- 图生视频

### 串行执行顺序建议
1. LLM 批量生成文本
2. TTS 批量生成音频
3. 图像批量生成关键帧
4. 视频批量生成镜头
5. FFmpeg 合成

---

## 7.4 GPU 作业队列设计

建议实现一个简化版队列：

### 表结构
- `gpu_jobs`
  - id
  - task_id
  - model_family
  - priority
  - status
  - created_at

### 规则
- 同一时刻仅 1 个 job status = running
- 高优先级修复任务可插队
- 长任务超时后自动释放锁

---

## 7.5 资源预估

### LLM 阶段
负载特征：
- 显存中等
- 上下文长度敏感
- 适合批量文本任务集中执行

优化策略：
- 分阶段裁剪上下文
- 避免把整章历史都塞进去
- 使用 story bible 摘要而不是所有原文

### TTS 阶段
负载特征：
- GPU/CPU 均可承担一部分
- 多句可串行或小批次处理

优化策略：
- 按 speaker 分批
- 对重复台词缓存
- 优先生成主线对白

### 图像阶段
负载特征：
- 扩散模型显存占用高
- 单图任务可控

优化策略：
- 低分辨率起稿
- 满意后再选少数上采样
- 参考图缓存复用

### 视频阶段
负载特征：
- 最吃时长
- 最吃显存
- 失败成本高

优化策略：
- 480p 起步
- 单镜头短时长
- 先低成本预览
- 通过关键帧控制成功率

---

## 7.6 预设档位设计

建议定义预设，而不是让上层直接传复杂参数。

### LLM 预设
- `novel_fast`
- `novel_quality`
- `script_strict_json`

### 图像预设
- `char_ref_base`
- `keyframe_draft`
- `keyframe_final`

### 视频预设
- `i2v_preview_480p`
- `i2v_final_480p`
- `i2v_try_720p`

### 合成预设
- `preview_low_bitrate`
- `release_standard`

---

## 7.7 质量与速度的折中

v1 应优先保证：

- 结构稳定
- 角色一致
- 内容完整
- 可重做

不要优先保证：
- 极致分辨率
- 每镜头都复杂运动
- 高度动态镜头

### 推荐默认
- 480p
- 镜头 3–5 秒
- 低至中等运动强度
- 音频和叙事优先于画面炫技

---

## 7.8 失败降级策略

### LLM 失败
- 降低上下文长度
- 切换更严格 schema prompt
- 启用 JSON 修复链

### TTS 失败
- 切换备选 speaker config
- 关闭过强情绪标签
- 转人工指定发音

### 图像失败
- 降低复杂度 prompt
- 固定更强 reference
- 回退 seed / preset

### 视频失败
- 分辨率降级
- 帧数降级
- motion prompt 更保守
- 切换为静态镜头 + 缓慢推拉的兜底策略

---

## 7.9 缓存策略

推荐缓存维度：

### 文本缓存
- prompt hash
- context hash
- model version

### 图像缓存
- prompt
- seed
- reference set
- preset

### 视频缓存
- keyframe hash
- motion prompt hash
- preset
- fps / length

---

## 7.10 推荐硬件运行策略

对于单台 24GB 机器，建议：

- LLM、图像、视频不要并发
- 音频与 FFmpeg 可穿插在 CPU 阶段
- 优先把最重的阶段安排在夜间批处理
- 白天用于审片、修 prompt、重跑局部

---

## 7.11 未来升级空间

后续如果扩到双 GPU 或更大显存：
- GPU1 常驻 LLM
- GPU2 跑图像/视频
- 吞吐会显著改善

但 v1 不应依赖此条件。
