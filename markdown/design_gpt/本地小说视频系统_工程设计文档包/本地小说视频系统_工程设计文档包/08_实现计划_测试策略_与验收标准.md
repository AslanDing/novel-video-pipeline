# 08 实现计划、测试策略与验收标准

## 8.1 开发阶段规划

建议分 6 个迭代阶段。

### Phase 0：基础设施
目标：
- 项目目录初始化
- 配置加载
- 数据库 schema
- 日志系统
- 基础 CLI / API

交付：
- 可创建项目
- 可写入元数据
- 可查询状态

### Phase 1：文本闭环
目标：
- LLM worker 接入
- 生成 story bible / character bible / chapter / script

交付：
- 从构思到 script.jsonl 的闭环

### Phase 2：语音闭环
目标：
- TTS worker 接入
- line 级音频生成
- 时长统计
- ASR QC

交付：
- 从 script 到对白与旁白音频闭环

### Phase 3：图像闭环
目标：
- 角色参考图生成
- 关键帧生成
- 关键帧元数据落盘

交付：
- 从 script 到 keyframes 闭环

### Phase 4：视频闭环
目标：
- I2V worker 接入
- shot 视频生成
- shot 局部重跑

交付：
- 从 keyframe 到 shot videos 闭环

### Phase 5：合成与发布
目标：
- FFmpeg 合成
- 字幕生成
- render report
- 一键导出

交付：
- 整章成片可导出

---

## 8.2 里程碑定义

### M1
从构思到结构化脚本

### M2
从脚本到全量音频

### M3
从脚本到关键帧

### M4
从关键帧到镜头视频

### M5
从所有镜头到最终成片

### M6
支持局部重跑与断点恢复

---

## 8.3 测试分层

## 单元测试
覆盖：
- schema 校验
- prompt 构造器
- 目录生成器
- artifact 版本切换
- 状态机转换

## 集成测试
覆盖：
- orchestrator 调 worker
- worker 结果回写数据库
- 局部重跑链路
- 断点恢复

## 端到端测试
覆盖：
- 从 concept 到 final.mp4 的完整链路
- 一章样例项目

## 回归测试
覆盖：
- 模型版本更新
- prompt 模板更新
- worker 参数更新

---

## 8.4 测试数据集建议

建议准备 3 类样例项目：

### Case A：短篇单人叙事
特点：
- 旁白为主
- 对白少
- 镜头简单

### Case B：双人对话驱动
特点：
- 对白密集
- 角色切换频繁
- 便于测试多 speaker 和 shot 拆分

### Case C：多场景剧情推进
特点：
- 角色 3–5 个
- 场景切换
- 有动作镜头

---

## 8.5 验收指标

### 文本阶段
- story bible schema 合法率 >= 95%
- script.jsonl schema 合法率 >= 95%
- 单章 script 无空镜头占比 >= 98%

### TTS 阶段
- 单句生成成功率 >= 95%
- ASR 文本相似度平均 >= 0.90
- 单句可重跑成功率 >= 95%

### 图像阶段
- 关键帧生成成功率 >= 90%
- 同角色一致性人工抽样可接受率 >= 80%

### 视频阶段
- shot 视频生成成功率 >= 80%
- 经一次自动降级后最终可用率 >= 90%

### 合成阶段
- 成片导出成功率 >= 95%
- 字幕时间轴错误率 <= 2%

---

## 8.6 质量红线

若出现以下情况，版本不可发布：

- 无法局部重跑
- 任务状态不一致
- active artifact 丢失
- 合成结果时间轴错乱
- 项目恢复后重复覆盖资产
- 角色在同场景内严重漂移

---

## 8.7 日志与调试建议

每个 task run 建议记录：
- task_id
- trace_id
- worker_name
- 输入摘要
- 配置快照
- 开始结束时间
- 输出文件列表
- 错误堆栈

建议目录：
- `logs/orchestrator.log`
- `logs/workers/llm/*.log`
- `logs/workers/tts/*.log`
- `logs/workers/image/*.log`
- `logs/workers/video/*.log`

---

## 8.8 自动化测试建议

建议把以下流程加入 CI：

1. schema 测试
2. workflow dry-run 测试
3. 文件命名规范测试
4. task 状态迁移测试
5. rollback 测试

模型推理本身可做 mock，避免 CI 太重。

---

## 8.9 人工评审清单

在每个里程碑做一次人工评审：

### 文本评审
- 角色设定是否自洽
- 分镜是否过长
- 对白是否像人说的话

### 语音评审
- 角色区分度
- 发音准确率
- 语气自然度

### 图像评审
- 角色是否稳定
- 构图是否符合镜头意图
- 风格是否统一

### 视频评审
- 运动是否自然
- 是否出现明显闪烁、坍塌
- 是否匹配音频时长

---

## 8.10 发布建议

v1 发布时建议只承诺：
- 单机
- 单项目
- 单章
- 480p
- 旁白 + 多人对白 + 动态分镜成片

不要在 v1 承诺：
- 高时长长视频
- 复杂口型同步
- 电影级动作场面
