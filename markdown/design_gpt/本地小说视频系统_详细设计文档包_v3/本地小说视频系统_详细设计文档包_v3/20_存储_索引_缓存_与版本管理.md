# 20_存储_索引_缓存_与版本管理

## 1. 设计目标

系统会产生大量中间产物。如果没有一套清晰的存储和缓存策略，很快就会出现：

- 找不到当前版本
- 磁盘爆炸
- 重复生成
- 无法比较不同结果
- 无法回滚

---

## 2. 存储类型

### 2.1 元数据存储
推荐 SQLite 起步，后续可切 PostgreSQL。
保存：
- 项目结构
- 任务状态
- artifact 元数据
- 版本信息
- 事件日志

### 2.2 文件存储
本地文件系统。
保存：
- 文本产物
- 音频
- 图片
- 视频
- 日志
- 配置快照

---

## 3. 路径规范

```text
storage/
  projects/{project_id}/
    episodes/{episode_id}/
      scripts/
      audio/
      keyframes/
      videos/
      subtitles/
      exports/
    shared/
      characters/
      voices/
      backgrounds/
  manifests/
```

---

## 4. Artifact manifest

每个 artifact version 都要有 manifest，例如：

```json
{
  "artifact_id": "a_001",
  "artifact_version_id": "av_003",
  "artifact_type": "shot_video",
  "scope_type": "shot",
  "scope_id": "SC01_SH03",
  "created_at": "...",
  "source_task_id": "task_332",
  "input_snapshot_hash": "abc123",
  "effective_config_hash": "def456",
  "files": [
    {"path": "shot_video.mp4", "sha256": "...", "size": 1234567}
  ]
}
```

---

## 5. 缓存策略

### 5.1 命中条件
同一能力、同一输入快照、同一有效配置时，可直接命中缓存。

### 5.2 缓存粒度
- LLM：story bible / outline / script / prompt
- TTS：句级音频
- Image：镜头级关键帧
- I2V：镜头级视频

### 5.3 缓存失效
当下列任一变化发生时，缓存失效：
- 输入文本变了
- 参考图变了
- 角色/声音 profile 版本变了
- effective config 变了

---

## 6. 清理策略

### 6.1 永久保留
- current versions
- 被手工批准的版本
- 最终导出版本

### 6.2 可归档
- 自动生成但未被选中的候选
- 失败任务产物
- 旧版预览视频

### 6.3 清理触发
- 磁盘使用超过阈值
- 项目归档
- 手工触发清理

---

## 7. 比较与 diff

系统应支持对两个版本的 artifact 做比较：

- 文本 diff
- 音频时长差异
- 图片缩略图对比
- 视频预览对比
- config diff

这对 prompt 调优很重要。

---

## 8. 版本选择策略

### 自动选择
- 根据 QC 分数自动选最高分

### 人工选择
- 用户可指定某个版本为 current

### 审批策略
- 若存在 manual_approved，则其优先级高于自动选择

---

## 9. 评审 checklist

- 是否有清晰路径规范
- 是否有 manifest
- 是否定义缓存命中规则
- 是否支持 current version 指针
- 是否有清理与归档策略
