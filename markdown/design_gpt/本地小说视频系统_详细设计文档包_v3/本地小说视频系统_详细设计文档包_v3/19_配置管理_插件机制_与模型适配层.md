# 19_配置管理_插件机制_与模型适配层

## 1. 目标

为了避免系统被具体模型绑死，需要建立三层解耦：

1. **业务意图**
2. **能力接口**
3. **模型实现**

例如：
- 业务意图：生成镜头关键帧
- 能力接口：`image.generate_keyframe`
- 模型实现：SD3.5 / FLUX / 其他

---

## 2. 配置分层

### 2.1 全局配置
- 路径
- 默认分辨率
- 队列配置
- GPU 策略
- FFmpeg 配置

### 2.2 模板配置
- 某个 workflow 的默认策略
- 不同模板的 prompt 版本
- 默认候选数

### 2.3 项目配置
- 风格
- 输出时长
- 角色绑定
- 优先使用的模型

### 2.4 任务配置
- 当次任务的覆盖项
- 降级策略
- 重试次数

---

## 3. 插件接口

### 3.1 LLMPlugin
```python
class LLMPlugin(Protocol):
    def run(self, task: LLMTask) -> LLMResult: ...
```

### 3.2 TTSPlugin
```python
class TTSPlugin(Protocol):
    def synthesize(self, segment: TTSegment, profile: VoiceProfile) -> TTSResult: ...
```

### 3.3 ImagePlugin
```python
class ImagePlugin(Protocol):
    def generate_keyframe(self, request: ImageTask) -> ImageResult: ...
```

### 3.4 I2VPlugin
```python
class I2VPlugin(Protocol):
    def generate_video(self, request: I2VTask) -> I2VResult: ...
```

---

## 4. 能力注册表

建立 `capability_registry`：

```yaml
llm.generate_story_bible:
  plugin: qwen_vllm
llm.generate_script:
  plugin: qwen_vllm
tts.synthesize_segment:
  plugin: fish_speech
image.generate_keyframe:
  plugin: sd35_comfyui
i2v.generate_shot_video:
  plugin: hunyuan_i2v
```

好处：
- workflow 不直接绑定模型
- 更换模型只改注册表
- 可按项目切换实现

---

## 5. 模型适配层职责

- 统一输入输出协议
- 参数映射
- 路径与格式转换
- 错误码标准化
- metrics 收集

例如 SD3.5 和 FLUX 接受的参数不同，但业务层只关心：
- prompt
- negative_prompt
- reference_images
- seed
- candidate_count

适配层负责翻译成底层调用参数。

---

## 6. 错误码规范

建议统一：
- `E_INPUT_INVALID`
- `E_RESOURCE_OOM`
- `E_MODEL_TIMEOUT`
- `E_MODEL_CRASH`
- `E_FORMAT_INVALID`
- `E_QC_FAILED`
- `E_DEPENDENCY_MISSING`

适配层必须把底层模型的错误统一映射为系统错误码。

---

## 7. 配置读取优先级

`task override > project config > workflow template config > global default`

必须明确，避免排查困难。

---

## 8. 配置文件组织

```text
config/
  app.yaml
  workers.yaml
  workflows/
  models/
    llm.yaml
    tts.yaml
    image.yaml
    i2v.yaml
```

---

## 9. 运行时配置快照

任何任务执行时，都要保存一份 `effective_config_snapshot.json`。
好处：
- 结果可复现
- 便于排查
- 便于比较不同参数版本

---

## 10. 评审 checklist

- 是否实现业务能力与模型实现解耦
- 是否有 capability registry
- 是否有统一错误码
- 是否有配置优先级规则
- 是否保存 effective config snapshot
