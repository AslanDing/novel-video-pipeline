# 23_前后端交互与任务中心设计

## 1. UI 目标

前端不是简单上传页面，而是“任务中心 + 产物浏览器 + 调试面板”。用户必须能：

- 创建项目
- 选择 workflow 模板
- 绑定角色音色和视觉参考
- 发起生成
- 查看进度
- 选择版本
- 重跑某个节点
- 导出成片

---

## 2. 核心页面

### 2.1 项目创建页
输入：
- 构思
- 风格
- 目标章节数
- 目标视频时长
- workflow 模板

### 2.2 项目总览页
展示：
- 项目状态
- 最近事件
- 当前章节进度
- 失败节点列表

### 2.3 章节工作台
展示：
- scene / shot 树
- 每个 shot 的关键帧、视频、音频状态
- 支持局部重跑

### 2.4 版本浏览页
展示：
- 当前 artifact 版本
- 历史版本
- diff / 预览 / 切换 current

### 2.5 调试页
展示：
- effective config
- prompt 模板版本
- worker metrics
- 错误日志

---

## 3. 交互原则

- 任何长任务都返回 task id，不做同步等待
- 所有资源都可预览
- 所有失败都要能点开看到原因
- 所有人工操作都形成事件写回系统

---

## 4. 典型交互流程

### 4.1 重跑一个镜头
1. 用户打开 shot 详情
2. 查看关键帧候选
3. 切换到另一个关键帧版本
4. 点击“从视频阶段重跑”
5. UI 发命令到 API
6. Orchestrator 生成新任务
7. UI 通过轮询或 SSE 刷新状态

### 4.2 修复一条台词
1. 用户在 TTS 列表里找到句子
2. 选择重跑 / 替换 voice profile
3. 重新生成
4. 系统自动标记下游混音 superseded

---

## 5. API 边界

前端只调用业务 API，不直接调用 worker。

核心 API：
- `POST /projects`
- `GET /projects/{id}`
- `POST /projects/{id}/commands/start`
- `POST /shots/{id}/commands/retry`
- `POST /artifacts/{id}/commands/select-version`
- `GET /tasks/{id}`
- `GET /episodes/{id}/timeline`

---

## 6. 状态展示建议

每个节点统一显示：
- 状态
- 耗时
- 当前版本
- 是否 manual approved
- 最近错误码

---

## 7. 评审 checklist

- 是否支持项目级、章节级、镜头级操作
- 是否支持版本选择和切换
- 是否支持局部重跑
- 是否有调试页和日志入口
- 是否避免前端直接耦合 worker
