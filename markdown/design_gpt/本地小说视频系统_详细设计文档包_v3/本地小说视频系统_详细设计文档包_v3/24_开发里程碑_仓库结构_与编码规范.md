## 3. 编码规范

### 3.1 目录规范
- 领域对象与服务逻辑分离
- worker 适配层与业务层分离
- 所有 schema 独立包管理

### 3.2 命名规范
- 任务类型统一用 snake_case
- artifact type 统一枚举
- API 路由统一复数名词 + commands

### 3.3 错误处理
- 不允许吞异常
- 必须返回标准错误码
- worker 异常必须带 stderr artifact

### 3.4 文档规范
- 每个模块 README
- 每个接口有 schema 示例
- 每个 worker 有运行说明

## 6. Definition of Done

一个模块完成，不是“能跑”就行，而是要满足：
- 有输入输出 schema
- 有错误码
- 有最小测试
- 有日志
- 有 README
- 能接入 Orchestrator
- 支持至少一条典型路径
