# 22_测试矩阵_故障注入_与性能基线

## 1. 测试目标

本系统的测试不能只做单元测试。必须覆盖：

- 功能正确性
- 状态一致性
- 产物完整性
- 失败恢复
- 性能与资源约束
- 局部重跑正确性

---

## 2. 测试层次

### 2.1 单元测试
适用模块：
- parser
- validator
- config resolver
- timeline builder
- idempotency key builder

### 2.2 集成测试
适用链路：
- API -> Orchestrator -> Worker -> Artifact
- 版本切换 -> 下游 superseded
- shot 级重跑

### 2.3 端到端测试
适用场景：
- 从构思生成 1 个最小章节
- 全链路成片
- 部分失败后恢复

---

## 3. 测试矩阵

| 维度 | 测试项 |
|---|---|
| 输入 | 极短构思 / 极长构思 / 缺字段 / 非法字段 |
| LLM | JSON 输出异常 / 长上下文裁剪 / 一致性冲突 |
| TTS | voice profile 缺失 / 发音错误 / 时长超限 |
| 图像 | 参考图缺失 / 构图不符 / 多余肢体 |
| I2V | OOM / 严重漂移 / 解码失败 |
| 合成 | 字幕错位 / 视频缺失 / FFmpeg 命令失败 |
| 状态 | 幂等重试 / superseded / manual approval |
| 存储 | 磁盘不足 / 文件损坏 / manifest 缺失 |

---

## 4. 故障注入点

### 4.1 Worker 层
- 返回 500
- 超时
- 返回格式错误
- 生成空文件
- 生成损坏文件

### 4.2 Orchestrator 层
- 任务重复派发
- 事件重复消费
- 状态未提交成功

### 4.3 存储层
- 文件写入失败
- 校验和不匹配
- 磁盘空间不足

---

## 5. 性能基线

你需要建立一套“参考项目基线”，例如：
- 1 章
- 3 个场景
- 12 个镜头
- 20 条语音 segment

然后记录：
- 总耗时
- LLM 耗时
- TTS 耗时
- 关键帧耗时
- I2V 耗时
- 最终导出耗时
- 峰值显存
- 磁盘占用

之后任何优化或回归都基于这个基线比较。

---

## 6. 验收阈值建议

### v1
- 单章最小闭环成功率 >= 80%
- 镜头级重跑正确率 >= 95%
- 无人工修复情况下导出成功率 >= 70%
- 所有任务幂等可复现

### v2
- 关键帧自动通过率提升
- I2V 重跑率下降
- TTS ASR 一致率更高

---

## 7. 测试数据集

建议自己维护小型内部测试集：
- 旁白密集型章节
- 对白密集型章节
- 多角色出场章节
- 强动作场景
- 情绪平缓场景

---

## 8. 评审 checklist

- 是否覆盖单元/集成/E2E
- 是否有故障注入点
- 是否有性能基线
- 是否有明确验收阈值
- 是否覆盖状态一致性和局部重跑
