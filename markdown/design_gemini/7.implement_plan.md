## 关键更新：蓝图生成拆分 (P0 - 最高优先级) ⚠️

### 问题识别

当前 `_create_blueprint` 一次性生成完整蓝图，包含：
- `world_building` - 世界观设定
- `characters` - 角色列表 (1-10个)
- `plot_structure` - 情节结构
- `chapter_plans` - 章节规划 (可能10-50章)

**问题**:
1. 单次调用 token 消耗大
2. 生成质量不稳定，LLM 容易偷懒省略细节
3. 无法针对特定部分进行重试/优化
4. Prompt 过长导致某些部分被压缩

### 拆分方案

```python
async def _create_blueprint(self, concept: NovelConcept) -> StoryBlueprint:
    """分步创建故事蓝图"""

    # Step 1: 生成世界观
    world_building = await self._generate_world_building(concept)

    # Step 2: 生成角色 (基于世界观)
    characters = await self._generate_characters(concept, world_building)

    # Step 3: 生成修炼体系 (基于世界观)
    power_system = await self._generate_power_system(concept, world_building)

    # Step 4: 生成情节结构 (基于角色和世界观)
    plot_structure = await self._generate_plot_structure(concept, world_building, characters)

    # Step 5: 生成章节规划 (批量生成)
    chapter_plans = await self._generate_chapter_plans(concept, world_building, characters, plot_structure)

    return StoryBlueprint(...)
```

### 需要新增的 Pydantic 模型

```python
class WorldBuildingOutput(BaseModel):
    """世界观输出模型"""
    setting: str = Field(description="世界描述，涵盖历史、地理和当前局势", min_length=200)
    factions: List[Dict[str, str]] = Field(description="主要势力列表")
    rules: List[str] = Field(description="世界运行的核心规则")

class CharactersOutput(BaseModel):
    """角色列表输出模型"""
    characters: List[CharacterSchema] = Field(min_length=1, max_length=10)

class PowerSystemOutput(BaseModel):
    """修炼体系输出模型"""
    power_system: str = Field(description="修炼体系、力量等级划分")
    cultivation_realms: List[Dict[str, str]] = Field(description="修炼境界列表")

class PlotStructureOutput(BaseModel):
    """情节结构输出模型"""
    plot_structure: List[PlotPointSchema] = Field(description="主线剧情结构")

class ChapterPlansOutput(BaseModel):
    """章节规划输出模型"""
    chapter_plans: List[ChapterPlanSchema] = Field(description="各章节细化规划")
```

#### 0.2 新增拆分用的提示词模板

- **新增模板**:

  - `CHARACTERS_PROMPT` - 角色生成提示
  - `POWER_SYSTEM_PROMPT` - 修炼体系生成提示
  - `PLOT_STRUCTURE_PROMPT` - 情节结构生成提示
  - `CHAPTER_PLANS_PROMPT` - 章节规划生成提示
- **工作量**: 0.5天

#### 0.3 修改 _create_blueprint 为分步调用
- **文件**: `stages/stage1_novel/novel_generator.py`
- **修改内容**:
  - 将 `_create_blueprint` 改为 async 方法
  - 分5步调用 LLM，每步使用对应的 Output 模型
  - 每步独立验证和重试
  - 保存中间结果到 `data/` 目录供调试
- **工作量**: 2-3天

#### 0.4 优化章节规划批量生成
- 对于章节数 > 10 的小说，使用批量生成
- 每批 5 章，分批生成
- 确保不遗漏任何章节
- **工作量**: 1天
