# AI小说视频生成系统 - 本地简化版设计概览

## 项目目标

构建一个纯本地化运行的小说到视频生成系统，无需部署，无需复杂服务架构，专注于功能实现和缓存机制，便于调试和迭代。

## 核心设计原则

1. **纯本地运行**: 所有模型本地加载，无需外部API（除可选LLM外）
2. **无Web界面**: 命令行操作，专注功能实现
3. **完整缓存机制**: 每个阶段都有缓存，支持断点续传
4. **便于调试**: 详细的日志和中间产物保存
5. **四阶段流程**: 保持原有 stage 设计

## 技术选型配置

### Stage 1: 小说生成 (LLM)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **模型** | Qwen3-14B-Instruct-AWQ | 14B量级，适合24GB单卡 |
| **推理框架** | vLLM | 高效推理，OpenAI兼容API |
| **上下文长度** | 32K | 支持长上下文小说生成 |
| **量化** | AWQ INT4 | 降低显存占用 |
| **温度** | 0.7 | 创造性vs确定性平衡 |

**备选模型**: Qwen2.5-14B-Instruct、Yi-1.5-9B-Chat

### Stage 2: 图像生成 (Image Generation)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **基础模型** | Stable Diffusion 3.5 Medium | 高质量图像生成 |
| **推理框架** | diffusers | HuggingFace官方框架 |
| **分辨率** | 1024x1024 | SD3.5原生分辨率 |
| **步数** | 30-50 | 质量vs速度平衡 |
| **CFG Scale** | 7.0 | 提示词遵循度 |

**角色一致性控制**:

| 技术 | 用途 | 配置 |
|------|------|------|
| **IP-Adapter** | 角色面部一致性 | 权重: 0.6-0.8 |
| **ControlNet** | 姿态/构图控制 | OpenPose + Canny |
| **Reference-only** | 风格统一 | 启用 attention耦合 |

**备选模型**: SDXL 1.0、Flux.1-schnell

### Stage 3: 音频生成 (TTS)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **主TTS引擎** | Fish Audio S2 Pro | 高质量多角色配音 |
| **备选TTS** | CosyVoice 3.0 | 中文旁白优化 |
| **保底方案** | Edge-TTS | 无需显存，多音色 |
| **采样率** | 44100 Hz | CD音质 |
| **格式** | WAV / MP3 | 无损压缩兼顾 |

**角色音色映射 (Voice Bible)**:

```json
{
  "narrator": {
    "engine": "fish-audio",
    "voice_id": "calm_narrator_v1",
    "style_tags": ["calm", "cinematic"],
    "speed": 1.0
  },
  "protagonist_male": {
    "engine": "fish-audio",
    "voice_id": "hero_young_male_v2",
    "style_tags": ["energetic", "heroic"],
    "speed": 1.05
  }
}
```

**备选TTS**: ChatTTS、GPT-SoVITS

### Stage 4: 视频合成 (Video Generation)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **视频生成模型** | HunyuanVideo-1.5 I2V | 腾讯开源图生视频 |
| **分辨率** | 480p (854x480) | 24GB单卡友好 |
| **帧率** | 24 fps | 电影标准 |
| **生成时长** | 3-5秒/镜头 | 短镜头保证稳定性 |
| **运动强度** | 中等 (motion_bucket=127) | 平衡动感和稳定 |

**备选方案 - Ken Burns效果**:

当显存不足或I2V失败时，使用FFmpeg实现静态图动态效果：

```bash
# 缓慢推镜头 (Slow push-in)
ffmpeg -i image.png -vf "zoompan=z='min(zoom+0.0015,1.5)':d=125:x='iw/2-(iw/zoom/2)':y='ih/2-(ih/zoom/2)'" output.mp4
```

**视频合成参数**:

| 参数 | 值 | 说明 |
|------|-----|------|
| 编码器 | H.264 (libx264) | 兼容性最佳 |
| 预设 | medium | 速度/质量平衡 |
| CRF | 23 | 视觉无损 |
| 像素格式 | yuv420p | 兼容性 |

### 备选技术栈总结

| 阶段 | 首选方案 | 备选方案1 | 备选方案2 |
|------|----------|-----------|-----------|
| **LLM** | Qwen3-14B-AWQ | Qwen2.5-14B | Yi-1.5-9B |
| **图像** | SD 3.5 Medium + IP-Adapter | SDXL 1.0 | Flux.1-schnell |
| **TTS** | Fish Audio S2 Pro | CosyVoice 3.0 | Edge-TTS |
| **视频** | HunyuanVideo-1.5 I2V 480p | SVD-XT | Ken Burns (FFmpeg) |

## 四阶段详细说明

### Stage 1: 小说生成 (Novel Generation)
- **输入**: 主题、类型、章节数等参数
- **输出**: `outputs/novels/{name}/novel.json`
- **功能**:
  - 角色设定生成
  - 世界观构建
  - 章节大纲生成
  - 正文内容生成
  - 分镜描述提取
- **缓存**: 每章独立缓存，支持断点续写

### Stage 2: 图像生成 (Image Generation)
- **输入**: Stage 1 的小说JSON
- **输出**: `outputs/images/{chapter}/{scene}.png`
- **功能**:
  - 场景描述提取
  - Prompt生成和优化
  - 角色一致性控制 (IPAdapter/Regional Prompting)
  - 图像生成 (SDXL/Flux)
- **缓存**: 已生成的图像跳过，支持重新生成指定场景

### Stage 3: 音频生成 (Audio Generation)
- **输入**: Stage 1 的小说JSON
- **输出**: `outputs/audio/{chapter}/{segment}.mp3`
- **功能**:
  - 文本分段处理
  - 角色语音映射
  - TTS生成 (Edge-TTS/ChatTTS)
  - 音频合并
- **缓存**: 已生成的音频段跳过

### Stage 4: 视频合成 (Video Composition)
- **输入**: Stage 2 的图像 + Stage 3 的音频
- **输出**: `outputs/videos/{chapter}.mp4`
- **功能**:
  - 图像序列和音频对齐
  - 视频合成 (FFmpeg)
  - 字幕叠加
  - 转场效果
- **缓存**: 已合成的视频跳过

## 缓存机制设计

### 缓存层级

```
┌─────────────────────────────────────────────────────────────┐
│                      缓存架构                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   L1: 内存缓存 (运行时)                                      │
│   ├── LLM响应缓存                                           │
│   └── 频繁访问的配置                                        │
│                                                             │
│   L2: 磁盘缓存 (项目目录)                                     │
│   ├── cache/llm/          - LLM请求/响应                    │
│   ├── cache/images/       - 生成的图像元数据                 │
│   ├── cache/audio/        - 音频生成记录                     │
│   └── cache/metadata/     - 流程状态                        │
│                                                             │
│   L3: 输出缓存 (outputs/)                                    │
│   ├── novels/             - 生成的小说JSON                   │
│   ├── images/             - 生成的图像                       │
│   ├── audio/              - 生成的音频                       │
│   └── videos/             - 合成的视频                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 缓存键设计

```python
# LLM缓存键
"llm:{model}:{hash(prompt+params)}"

# 图像缓存键
"img:{chapter_idx}:{scene_idx}:{hash(prompt)}:{seed}"

# 音频缓存键
"audio:{chapter_idx}:{segment_idx}:{voice_id}:{hash(text)}"

# 视频缓存键
"video:{chapter_idx}:{mode}:{hash(inputs)}"
```

## 运行方式

### 完整流程

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置环境
cp .env.example .env
# 编辑 .env 设置本地模型路径

# 3. 运行完整流程
python main.py --novel "我的修仙小说" --genre 修仙 --chapters 3 --scenes-per-chapter 3
```

### 分阶段运行

```bash
# Stage 1: 小说生成
python run_stage1.py --novel "书名" --genre 修仙 --chapters 3

# Stage 2: 图像生成
python run_stage2.py --novel-dir "outputs/novels/书名"

# Stage 3: 音频生成
python run_stage3.py --novel-dir "outputs/novels/书名"

# Stage 4: 视频合成
python run_stage4.py --novel-dir "outputs/novels/书名" --mode compose
```

### 调试模式

```bash
# 启用详细日志
python main.py --novel "书名" --verbose --debug

# 只运行到指定阶段
python main.py --novel "书名" --stage 2

# 强制重新生成（忽略缓存）
python main.py --novel "书名" --force
```
