## 关键问题分析

### 问题1: 蓝图生成 Prompt 过于聚合 ⚠️⚠️⚠️ (P0)

**严重程度**: 高

**问题描述**:
当前 `_create_blueprint` 一次性生成完整的蓝图，包含世界观、角色、情节结构、章节规划，Prompt 内容过多，导致：
1. 生成质量不稳定
2. 某些字段被压缩省略
3. 难以精细控制各部分

**已解决的历史问题** (✅ 已实现):
| 项目 | 当前状态 | 状态 |
|------|---------|------|
| **Pydantic模型** | `stages/stage1_novel/pydantic_models.py` 使用 Pydantic v2 | ✅ 已实现 |
| **JSON Schema** | 通过 `model_json_schema()` 生成 | ✅ 已实现 |
| **response_format** | 在 `generate()` 调用时使用 `{"type": "json_object"}` | ✅ 已实现 |
| **协议式Prompt** | `stages/stage1_novel/prompts/protocol_prompts.py` 完整实现 | ✅ 已实现 |

**当前代码实现** (`novel_generator.py:151-224`):
```python
# 使用 Pydantic 模型 + 协议式 Prompt
system_prompt = generate_protocol_prompt(STORY_ARCHITECT_CORE, StoryBlueprintOutput)

response = await self.llm_client.generate(
    prompt=task_description,
    system_prompt=system_prompt,
    response_format={"type": "json_object"}
)

validated_output = StoryBlueprintOutput.model_validate(raw_data)
```

**需要改进的方向** - 拆分蓝图生成:
1. **世界观生成** (WorldBuildingOutput) - 单独生成
2. **角色设计** (CharactersOutput) - 单独生成
3. **体系设计** (PowerSystemOutput) - 单独生成
4. **情节结构** (PlotStructureOutput) - 单独生成
5. **章节规划** (ChapterPlansOutput) - 批量生成

### 问题2: 蓝图生成需要拆分 (P1)

**严重程度**: 中高

**问题描述**:
当前 `_create_blueprint` 一次性生成包含以下所有内容的完整蓝图：
- `world_building` - 世界观设定
- `characters` - 角色列表 (1-10个)
- `plot_structure` - 情节结构
- `chapter_plans` - 章节规划 (可能10-50章)

这导致：
1. 单次调用 token 消耗大
2. 生成质量不稳定，LLM 容易偷懒省略细节
3. 无法针对特定部分进行重试/优化

**建议的拆分方案**:
```
Step 1: 生成世界观 (WorldBuildingOutput)
Step 2: 生成角色 (CharactersOutput) - 基于世界观
Step 3: 生成修炼体系 (PowerSystemOutput) - 基于世界观
Step 4: 生成情节结构 (PlotStructureOutput) - 基于角色和世界观
Step 5: 生成章节规划 (ChapterPlansOutput) - 批量生成
```

### 问题3: 本地模型部署缺失 ⚠️ (P1)

**严重程度**: 中

**问题描述**:
设计文档要求**所有模型本地部署**，但 Stage 1 目前使用 NVIDIA NIM 云服务。

| 组件 | 设计要求 | 当前实现 | 状态 |
|------|---------|---------|------|
| **LLM (Stage 1)** | Qwen3-14B-AWQ via vLLM (本地) | NVIDIA NIM 云服务 | ❌ 未实现 |
| **图像生成 (Stage 2)** | SD 3.5 / SDXL (本地) | SDXL 本地 ✓ | ✅ 已实现 |
| **音频生成 (Stage 3)** | Fish Audio / CosyVoice (本地) | Edge-TTS (在线) | ⚠️ 降级实现 |
| **视频生成 (Stage 2)** | HunyuanVideo / SVD (本地) | 本地实现 ✓ | ✅ 已实现 |

**核心问题**: `core/local_llm_client.py` 已完整实现 OllamaClient 和 VLLMClient，但 `novel_generator.py` 仍然直接使用 `NVIDIA_NIM_Client`。

## 六、优先实现建议

基于以上分析，建议按以下优先级进行开发：

### P0 (最高优先级) - 蓝图生成拆分

1. **拆分蓝图生成 Prompt**
   - 影响：提高生成质量，减少 token 消耗，允许部分重试
   - 工作量：中等 (~3-5天)
   - 拆分步骤：
     - Step 1: 生成世界观 (WorldBuildingOutput)
     - Step 2: 生成角色 (CharactersOutput)
     - Step 3: 生成修炼体系 (PowerSystemOutput)
     - Step 4: 生成情节结构 (PlotStructureOutput)
     - Step 5: 生成章节规划 (ChapterPlansOutput)
   - 文件: `stages/stage1_novel/novel_generator.py`, `stages/stage1_novel/pydantic_models.py`

### P1 (高优先级) - 本地模型部署

2. **集成 LocalLLMClient 到 Stage 1**
   - 影响：核心架构要求 - 所有模型本地部署
   - 工作量：中等 (~2-3天)
   - 文件: `stages/stage1_novel/novel_generator.py`, `run_stage1.py`

### P2 (中优先级) - 体验优化

3. **Fish Audio / CosyVoice 集成**
4. **集成 CharacterPortraitManager 到 ImageGenerator**
5. **IPAdapter 完整集成**
6. **BGM/SFX 自动匹配**
